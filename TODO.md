# 1차 개발 검토 — 미구현/불일치 항목

> 검토 기준일: 2026-04-23  
> 참조 문서: 요구사항 명세서 v0.2, API 명세서 v1.1

---

## 🔴 즉시 결정 필요 (코드-문서 불일치)

### 1. GatheringStatus 값 이름 불일치
- **문서**: `RECRUITING(모집중)`, `CLOSED(마감)`, `COMPLETED(완료)`, `CANCELLED(취소)`
- **코드**: `OPEN` 사용 중 → `RECRUITING` 과 불일치
- **영향 범위**: `GatheringStatus.java`, `GatheringService`, 신청 가능 여부 판단 로직 (`status != GatheringStatus.OPEN`)
- **할 일**: 문서 기준으로 코드 수정 OR 문서를 코드 기준으로 업데이트 중 하나 결정

### 2. ApplicationStatus에 CONFIRMED 상태 (문서에 없음)
- **문서 FR-APP-07**: `PENDING`, `CANCELLED`, `ATTENDED` 3가지만 정의
- **코드**: `PENDING`, `CONFIRMED`, `CANCELLED`, `ATTENDED` 4가지 존재
- **할 일**: CONFIRMED를 정식 상태로 문서에 추가할지, 코드에서 제거할지 결정
  - 힌트: 관리자가 신청을 "확정"하는 중간 단계로 의도했을 가능성 있음

### 3. Application에 MBTI 필드 없음
- **문서 FR-APP-03**: "신청 시 성별, 나이, **MBTI**, 유입 경로를 입력한다"
- **코드**: `Application` 엔티티, `ApplicationRequest` DTO 모두 MBTI 필드 없음
- **할 일**: MBTI 추가 시 엔티티 + DTO + DB 컬럼 마이그레이션 모두 필요

## 🟡 미구현 — 2차 개발 대상 (범위 확인 필요)

### 4. 마일리지 시스템
- **문서**: FR-GTH-08 (참석 시 500 마일리지 지급), FR-USR-04 (마일리지 조회)
- **현황**: `ErrorCode.MILEAGE_NOT_ENOUGH` 는 이미 정의되어 있음 → 설계 의도는 있음
- **할 일**: 마일리지 도메인 전체 구현 (엔티티, 적립 로직, 조회 API)

### 5. 동물 아바타 (FR-USR-03)
- **문서**: 동물 종류, 색상, 포즈 선택 가능
- **현황**: `User` 엔티티에 아바타 관련 필드 없음
- **할 일**: 2차 개발 범위 확정 후 구현

### 6. 온보딩 프로필 초기 설정 (FR-USR-06)
- **문서**: 회원가입 직후 온보딩 과정 필요
- **현황**: 미구현
- **할 일**: 프론트 주도 기능인지, 백엔드 별도 API가 필요한지 확인

### 7. 모임 태그 — 무드/활동 태그 (FR-GTH-06)
- **문서**: 모임에 무드 태그와 활동 태그를 복수 설정 가능
- **현황**: `Gathering` 엔티티에 태그 관련 필드/연관관계 없음
- **할 일**: 태그 도메인 추가 설계 필요 (연결 엔티티 방식 권장)

### 8. 장소 계약 상태 (FR-LOC-04)
- **문서**: 장소의 계약 상태 관리 (계약중, 계약만료 등)
- **현황**: `LocationStatus` enum은 존재하나 `Location` 엔티티에 연결되어 있는지 불확실
- **할 일**: `Location` 엔티티 및 관련 API 확인 후 미연결이면 추가
